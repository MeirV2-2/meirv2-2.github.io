meirv2-2

I do low-level Windows research — internals, offensive technique work, reverse engineering. I write things up here when I learn something worth keeping a record of.

What I work on

  • Windows internals. The loader, the heap, the threadpool, undocumented bits of ntdll. The stuff under the API.
  • Offensive technique research. Injection, evasion, the cat-and-mouse with EDR memory scanners. What still works, what doesn't, and why.
  • Reverse engineering. Mostly Windows malware and system components when I'm trying to understand how a primitive is supposed to be used.
  • Detection engineering — from the attacker side. Building things specifically to see which scanners notice and which don't.
